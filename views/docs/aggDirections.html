agg-directions

The agg-directions directive provides methods and models for quickly creating a custom directions widget for your application. Unlike the agg-menu directive, this directive requires a template for it's display, thus allowing for complete customization of the directives style and layout.

In its simplest form, the template provided to this directive must contain 2 agg-auto-complete input boxes for taking in start and end locations as well as some markup for display of the generated directions

The directive uses controllerAs syntax with the namespace dir for the controller. Therefore, to call a method or access a model in your template, prefix the name with dir. For example, to assign the models to the origin input box you would write 

<input agg-auto-complete="dir.request.origin" id="origin" type="text" ng-model="dir.input.origin">

Requires the agg-map directive

Usage

<agg-map options="Object">
    <agg-directions template="path/to/template" method="String"></agg-directions>
</agg-map>

Attributes

Attribute Type Details
template String The path to the template to be used.
method String The type of directions to be returned. This optional attribute can have one of the following options or be omitted:
steps: this method will draw a route with markers at each step with text directions and fill every property of the route model,
simple: This method will draw a simple route with a marker at the start and end and fills the route models properties except for the steps[],
mapsteps: this method will draw a route with markers at each step with text directions but does NOT fill the route model,
default: if the method attribute is omitted or set to anything else only a simple route will be drawn on the map.

Models

Model Properties Details
input origin: String,
destination: String		 The input model is for attaching an ng-model directive to the two input boxes. The clear() method will clear this model out, subsequently clearing the input elements as well if this model is assigned to them.
request origin: Obj,
destination: Obj,
travelMode: String		 The request model's origin and destination properties are for the agg-auto-complete directives that should be attached to the input boxes. The travelMode property specifies which travelMode the directions will be returned as. Options for travelMode are: 'DRIVING', 'WALKING', 'BICYCLING', or 'TRANSIT'
route start: String,
end: String,
via: String,
distance: String,
duration: String,
steps: []		 This model stores the returned directions and is used to display the directions in the template. The properties start, end, via, distance, and duration all hold strings while the steps property is an array of DirectionsStep objects. For more information on the returned directions see the documentation for the DirectionsResult Object

Methods & Properties

Method Type Details
clear() void Clears out all the models, sets showDirect to false, and clears any markers and route from the map.
toggle() void Toggles the value of showDirect between true and false.
showDirect boolean A boolean value that can be used used with a directive like ng-if or ng-show to hide or display text directions. This value is toggled by the toggle() method and will be set to true anytime directions are successfully returned.
